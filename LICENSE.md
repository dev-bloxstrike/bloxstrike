2. **Use Silent Aim instead of hard-lock Aimbot** in public servers — snap-lock aimbot is visually obvious to spectators and kill-cams
3. **Keep smoothing values high** on the aimbot (70%+) — smooth tracking looks far more human than instant snapping
4. **Limit hitbox expansion to moderate values** — maxing the **bloxstrike script hitbox** slider makes kills look statistically impossible
5. **Use HvH Mode only in designated HvH lobbies** — running resolver logic in standard public matches is unnecessary and increases behavioral flags
6. **Update the script after every Bloxstrike patch** — an outdated anti-detection layer is worse than no layer at all
7. **Don't stream or record with obvious cheats visible** — clip sharing is a leading source of player reports
8. **Use private servers for testing new modules** before deploying them in competitive lobbies

## 🔧 Common Issues and Fixes

**❌ GUI doesn't appear after execution**
- Ensure Bloxstrike has fully loaded before injecting; executing during the loading screen prevents GUI initialization

**❌ Silent Aim not tracking targets**
- Open GUI settings and confirm "Target Bone" is set to **Head** or **Torso**; some executor versions default to a null value

**❌ ESP stops rendering mid-match**
- This occurs when the server refreshes character models; toggle ESP off and back on to reinitialize the render loop

**❌ HvH resolver isn't countering enemy anti-aim**
- Update to the latest script version; the resolver's anti-aim pattern database is updated with each release

**❌ Script fails to inject on mobile**
